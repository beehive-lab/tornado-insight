Flags methods that look like kernel entry points (annotated with @Parallel / @Reduce or taking a KernelContext parameter) but declare a non-void return type.

Kernels registered through TaskGraph.task(...) must return void: results are written into output buffers passed as parameters, not returned from the call. A non-void signature is legal only when the method is used purely as a helper that TornadoVM inlines into another kernel — which is why this inspection is reported as a warning rather than an error.

How to fix: change the return type to void and write the result into an output parameter (e.g. an IntArray / FloatArray), unless the method is only ever called from another kernel as a helper.
