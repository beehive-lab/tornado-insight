Flags exception-related constructs inside a TornadoVM kernel or any helper method that TornadoVM would inline:

  • throw statements.
  • try / catch blocks — the catch block is silently dropped and any failure in the try block goes unhandled on the device.
  • throws clauses declared on the kernel method or on a helper reachable from it.

GPUs and other accelerators do not have a traps mechanism that maps to Java exceptions. On a CPU, a fault such as integer division by zero sets a flag in a special register that the OS forwards to the JVM, which then raises an exception. GPUs have no such handoff (CUDA floating-point notes), so TornadoVM must guarantee that exceptional conditions never occur on the device. Any thrown exception, caught exception or declared throws would have nowhere to surface.

How to fix: validate the inputs in host code before calling TaskGraph.execute(), and write the kernel body so that every branch produces a defined result (e.g. clamp divisors, guard array indices).
