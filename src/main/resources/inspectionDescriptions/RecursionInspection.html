Flags direct or indirect recursive calls reachable from a TornadoVM kernel (a method annotated with @Parallel / @Reduce or one that takes a KernelContext parameter). The check follows callees across class boundaries.

Device backends used by TornadoVM — OpenCL, CUDA/PTX and SPIR-V — do not allow recursion: kernels run on hardware that has no managed call stack, so each call site must be statically inlinable.

How to fix: rewrite the algorithm iteratively (e.g. with an explicit loop or an explicit stack/queue backed by a TornadoVM array type).
