Flags Java assert statements that appear inside a TornadoVM kernel, i.e. a method annotated with @Parallel or @Reduce, or any helper method transitively inlined into one.

Assertions rely on the JVM to raise AssertionError and on the host to handle it. Code generated for OpenCL, PTX or SPIR-V runs on an accelerator that has no notion of Java exceptions, so the assertion would either be silently dropped or fail compilation.

How to fix: remove the assertion, or move the precondition check into host code that runs before TaskGraph.execute().
