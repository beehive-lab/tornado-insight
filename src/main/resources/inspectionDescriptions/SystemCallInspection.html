Flags two kinds of unsupported calls inside a TornadoVM kernel or any helper method that TornadoVM would inline:

  • Calls to native methods (JNI).
  • Calls into JDK system / IO / reflection classes that are restricted in kernel context (for example System, Thread, java.io.*, java.nio.*, java.lang.reflect.*).

A TornadoVM task can be dispatched to a device that does not share the host operating system or the JVM (a discrete GPU, an FPGA, an accelerator across PCIe). Code generated for OpenCL, PTX or SPIR-V cannot perform system calls, invoke JNI, or interact with JVM-managed resources.

How to fix: keep the call out of the kernel. Perform I/O, reflection and other host-only work before or after TaskGraph.execute(), and pass the resulting data into the kernel through TornadoVM array or vector types.
