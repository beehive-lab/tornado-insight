Flags variables, fields and parameters declared with a type that the TornadoVM compiler cannot translate to OpenCL, PTX or SPIR-V. The inspection runs on any kernel method (annotated with @Parallel or @Reduce) and on every helper method that TornadoVM would inline at runtime.

Supported types:

  • Java primitives: boolean, byte, short, char, int, long, float, double.
  • Primitive arrays of the above (e.g. int[], float[]).
  • TornadoVM API types under uk.ac.manchester.tornado.api.*: Panama-backed arrays such as IntArray and FloatArray, vector types such as VectorFloat4, and matrix types such as Matrix2DFloat.

TornadoVM knows the memory layout of these types and can generate specialised device code for them. For example, VectorFloat4 lowers to the OpenCL float4 vector type, which lets the compiler emit SIMD instructions on AVX-capable CPUs or vector lanes on GPUs.

The full list of TornadoVM data structures lives in the tornado-api module.

How to fix: replace the flagged type with one of the supported types above, or move the work that uses it out of the kernel into host code.
