Flags references to a static field of type uk.ac.manchester.tornado.api.TaskGraph, and calls to TaskGraph.task(...) on such a static instance.

TornadoVM lazily JIT-compiles the methods registered through task(...). If a TaskGraph lives in a static field, its initialization can run while the class is still being loaded; the TornadoVM compiler thread then needs the same class to finish initialization to compile the task, which can deadlock the user thread against the TornadoVM compiler thread.

How to fix: declare the TaskGraph as an instance field or a local variable, and create it after the surrounding class has finished initialization (for example inside a method).
