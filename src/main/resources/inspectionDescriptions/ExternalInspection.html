Flags method calls made from a TornadoVM kernel (or an inlined helper) into a class that is neither part of the JDK (java.*) nor of the TornadoVM API (uk.ac.manchester.tornado.*).

The TornadoVM JIT compiler can lower a curated subset of the JDK and the TornadoVM API to device code. Third-party libraries are out of scope: TornadoInsight cannot statically verify what they do, and at runtime TornadoVM will likely refuse to compile the kernel.

How to fix:

  • Inline the required functionality directly in the kernel using primitives and TornadoVM API types.
  • If the call has no equivalent on the device, move that work to host code that runs before or after the TaskGraph executes.
